# Security reporting

This repository is a software engineering portfolio project and is not operated
as a continuously monitored public service.

If you discover a security issue, please do not open a public issue containing
exploit details, credentials, personal information, or uploaded documents.
Contact the repository owner privately through the security-reporting channel
available on the Git hosting profile, and include:

- the affected component or route;
- reproduction steps using fictional data;
- the expected and observed behavior;
- the likely impact; and
- a suggested mitigation, if known.

Do not test against infrastructure or accounts you do not own or have explicit
permission to use. Do not upload real medical or personal information.

There is no guaranteed response-time SLA. Supported dependency versions and
deployment status should be verified from the current branch and CI rather than
assumed from an old release. Technical controls in this repository are not a
compliance certification.
